About Zulip

This server is an installation of the Zulip open source group chat software. Written in Python and using the Django web framework, Zulip has an extensive real-time messaging featureset, including both group and private messaging, conversation streams, powerful search, drag-and-drop file uploads, image previews, audible notifications, missed-message emails, markdown formatting, desktop and mobile apps, dozens of integrations, and much much more.

Zulip was designed from the ground up to optimize the producitivy of discussions and real-time decision-making. A key feature that drives this is Zulip's unique threading model and system for tracking unread messages, which make it natural to have multiple conversations in the same stream. As a result, Zulip is far more efficient than any other chat product for catching up on conversations while you were away from your devices: you can read exactly the threads you missed that are important to you, and it feels natural to follow up to conversations that happened while you were away.

Zulip's vision is to create the world's best group chat software, completely open source, so that everyone has the freedom to customize, improve, and run their own copy of this essential piece of collaboration infrastructure.

Further information on the Zulip project and its features can be found on the Zulip open source project's website.

Zulip Community

Zulip is developed by a vibrant community of developers from around the world. Every month, dozens of people contribute code to Zulip, and dozens more contribute bug reports, feedback, and translations. The Zulip community welcomes new contributors from any background. The Zulip project has an easy to install development environment, an extensive test suite, and over 50,000 words of developer documentation to make it easy for new contributors to contribute effectively to the project.

Contributing to Zulip

If you'd like to join the Zulip community and contribute to Zulip, please visit the main Zulip project on GitHub!

{% trans %}Zulip history{% endtrans %}

Zulip was originally developed by Zulip, Inc., a small startup in Cambridge, Massachusetts. Zulip, Inc. was founded by the MIT team that previously created the Ksplice powerful software for updating a running Linux kernel without rebooting. Zulip was inspired by the Barnowl client for the Zephyr protocol, and the incredible community that Zephyr supported at MIT.

Zulip, Inc. was acquired by Dropbox in early 2014, while the product was still only available in private beta. Dropbox eventually decided not to invest in the product. Zulip's beta users loved Zulip's unique user experience and continued enthusiastically using Zulip, despite the fact that Zulip was not being actively developed. Eventually, Dropbox decided to release Zulip as open source software so that Zulip's fanatical users could continue enjoying the software.

As a result, the first time the public had the opportunity to use Zulip was when Dropbox released Zulip as open source software in late 2015. Because of Zulip's powerful threading model and complete featureset, the Zulip open sourcing announcement was very popular, staying at the top of both Hacker News and the programming subreddit for an entire day.

Zulip was open sourced with the complete version control history intact because 10 Zulip users visited Dropbox for a full week to help with the technical work required to open source Zulip well. The Zulip community is incredibly grateful to both Dropbox and those enthusiastic early users of Zulip for making the Zulip open source project possible.

By the end of 2015, Zulip was already going strong with a community of dozens of developers around the world. As of mid-2016, more than 150 people from all over the world have contributed who have contributed almost 1000 improvements to the software, and the Zulip project is moving faster than when Zulip, Inc. had 11 full-time employees working on the Zulip software.
